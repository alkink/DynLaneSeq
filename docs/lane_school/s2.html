S2: Çizgi koordinatı artık görüntüden kanıt okumak için kullanılır.

S2’nin farkı şudur: Model sadece query ve row token ile tahmin yapmaz. Önce kaba çizgi üretir; sonra bu çizginin üstündeki feature’ları okur. Bu okunan şey evidence tensorüdür.

Evidence koordinat değildir. Evidence, çizgi üstündeki noktalardan okunan 256 sayılık feature vektörleridir: [B,20,72,256].

2. S2 Forward Akışındaki Tensorlar

# dynlaneseq_s2.py
enc = self.encoder.forward_features(images)
q = enc["queries"]                       # [B,20,256]
features = enc["features"]               # [B,256,72,200]

coarse_row = self.row_decoder(self.build_coarse_tokens(q))
coarse_x = coarse_row["pred_x_rows"]      # [B,20,72]

sample_x = self.curriculum.build_sample_x(coarse_x, targets, matches, alpha)
evidence = self.sampler(features, sample_x) # [B,20,72,256]

tokens = self.build_final_tokens(q, evidence)
row = self.row_decoder(tokens)
final_x = row["pred_x_rows"]              # [B,20,72]

Kod tarafı

S2 iki kez row decoder kullanır: önce kaba x için, sonra evidence eklenmiş final x için.

Lane tarafı

Önce mavi kesik çizgi çıkar. Sonra sarı noktalardan görüntü okunur. Sonra yeşil final çizgi çıkar.

3. coarse_x Tam Olarak Nedir?

coarse_x, S1’deki gibi row decoder’dan çıkan ilk x koordinatlarıdır.

coarse_x[b,n,p]

b = görüntü
n = lane adayı
p = y row
değer = o row'daki x piksel koordinatı

Örnek hayali değerler:

coarse_x[0,3,:] =
[310, 309, 307, ..., 210, 205, 199]

Bu liste tek başına bir lane çizgisidir. Ama S2 bunu final saymaz; bunu görüntüden kanıt okumak için kullanır.

4. sample_x Neden coarse_x ile Aynı Şey Değil?

Training başında model kötü çizebilir. Eğer doğrudan kötü coarse_x üstünden feature okunursa yanlış kanıt gelir. Bu yüzden curriculum devreye girer.

# sampler_curriculum.py
sample_x = coarse_x.clone()
mixed = alpha * gt_x + (1 - alpha) * pred
sample_x[b, pred_idx] = where(valid, mixed, pred)
sample_x = clamp(sample_x, 0, input_w - 1)

alpha=1

Sample noktası çoğunlukla gerçek lane üstüne konur. Öğretmen eli tutuyor.

alpha=0.5

Gerçek lane ve model tahmini karışır.

alpha=0

Model kendi tahmin ettiği çizgiden evidence okur.

Görselde sarı noktaların altındaki “görüntü kanıtı” alınır.

6. evidence[b,n,p,:] İçinde Ne Var?

Tek bir evidence vektörü şöyle okunur:

evidence[b,n,p,:]  # 256 sayı

b: hangi görüntü
n: hangi lane adayı
p: lane'in hangi y row'u
:  o noktadan okunan 256 kanallı feature

Bu 256 sayının içinde insanın okuyacağı şekilde “sarı çizgi var”, “gölge var” gibi etiketler yoktur. Ama eğitim sırasında model bu vektörleri lane kararında kullanmayı öğrenir.

7. Evidence Final Decoder’a Nasıl Ekleniyor?

# dynlaneseq_s2.py / build_final_tokens
row_emb = self.row_embedding.weight.view(1,1,72,256)
tokens = queries.unsqueeze(2) + row_emb + self.adapter(evidence)

queries.unsqueeze(2)

Hangi lane adayı olduğumuz bilgisi.

row_emb

Hangi y satırında olduğumuz bilgisi.

adapter(evidence)

Bu noktada görüntünün bize söylediği bilgi.

Bu üçü toplanınca final token oluşur: “Ben şu lane adayının şu row'uyum ve altımda şu görsel kanıt var.”

8. EvidenceAdapter Neden Var?

# evidence_adapter.py
self.gamma = learnable scalar
return self.gamma * self.net(evidence)

Evidence direkt eklenirse eğitim başında modeli bozabilir. gamma küçük başlar. Bu, evidence etkisini kademeli açılan bir musluk gibi düşün.

gamma küçükse

Model S1’e yakın davranır. Evidence az etki eder.

gamma büyürse

Model lane üstünden okuduğu görüntü bilgisini daha çok kullanır.

9. Güncel S2: Active Corridor ile Sağ-Sol Piksel Arama

Güncel S2/S3'te evidence sadece kaba çizginin tam üstünden okunmayabilir. ActiveCorridorSearch kaba x'in sağında ve solunda küçük bir koridor açar.

# dynlaneseq_s2.py
offsets = [-32,-24,-16,-8,0,8,16,24,32]
offset_samples = sampler.sample_local_window(features, center_x)
# [B,20,72,9,256]

evidence, pred_delta, weights = active_corridor(offset_samples, q)
refined_x = center_x + pred_delta

weights[b,n,p,o] şu sorunun cevabıdır: “n’inci lane’in p’inci row'unda, o offset noktasına ne kadar güveniyorum?” Bu yüzden S2 artık sadece “çizgi üstünden oku” değil, “çizgi çevresinde küçük arama yap” haline geldi.

10. Güncel S2: Multi-Scale Sampler Ne Ekliyor?

Feature map tek çözünürlükten gelmek zorunda değil. MultiScaleCurveAlignedSampler aynı lane noktasını p2, p3, p4 gibi farklı FPN seviyelerinden okur ve query+row bilgisiyle hangisine daha çok güveneceğini seçer.

sampled = [
  sample(p2, sample_x),  # daha detaylı, daha lokal
  sample(p3, sample_x),  # orta
  sample(p4, sample_x),  # daha geniş bağlam
]
gate = softmax(gate_mlp(query + row_embedding))
evidence = sum(gate * sampled)

p2

Yakın detay. Şerit boyasının ince kenarını daha iyi taşıyabilir.

p3

Denge. Hem detay hem bağlam.

p4

Geniş bağlam. Yolun genel yönünü daha iyi taşıyabilir.
