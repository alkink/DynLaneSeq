S1 derin anlatımQuery → 72 boncuk

S1: Bir query vektörü, 72 row token’a açılır.

S1’in farkı şudur: S0’da query tek MLP ile 72 row'u birden çıkarıyordu. S1’de query önce 72 ayrı token’a bölünür; her token çizginin bir yatay satırdaki parçasını temsil eder.

Row token koordinat değildir. Row token [256] boyutlu bir temsil vektörüdür. Koordinat yine row_x_logits → soft_expected_x ile çıkar.

1. S1’in Görsel Karşılığı

Bu görselde sarı noktaları “row token” diye düşünmek iyi bir başlangıçtır; ama teknik olarak row token nokta değildir. Row token, o noktanın kararını verecek 256 sayılık vektördür.

2. S1’de Hangi Tensorlar Var?

# dynlaneseq_s1.py
enc = self.encoder.forward_features(images)
q = enc["queries"]                 # [B,20,256]
row_emb = self.row_embedding.weight # [72,256]
tokens = q.unsqueeze(2) + row_emb   # [B,20,72,256]

q

20 lane adayı. Her aday 256 sayı. Henüz row bilgisi yok.

row_emb

72 satırın öğrenilen kimliği. “Ben alt row muyum, üst row muyum?” bilgisini taşır.

tokens

Her lane adayı, 72 row’a kopyalanmış ve row kimliği eklenmiş hali.

3. Broadcasting Tam Olarak Ne Yapıyor?

Bu satır kritik:

tokens = queries.unsqueeze(2) + row_emb

Shape dönüşümü:

queries              = [B,20,256]
queries.unsqueeze(2)= [B,20,1,256]
row_emb             = [1,1,72,256]
tokens              = [B,20,72,256]

Array olarak

tokens[b,n,p,:] = b’inci görüntüde, n’inci lane adayı, p’inci yatay satır için 256 sayılık vektör.

Bu vektörün içinde açık x koordinatı yoktur.

Lane üzerinde

Bir lane çizgisini 72 küçük karar noktasına bölmüş olduk.

Her karar noktası “ben bu y satırında nereye oturacağım?” sorusunu cevaplayacak.

4. RowTokenDecoder Ne Yapıyor?

# row_token_decoder.py
b, n, p, d = row_tokens.shape       # p=72, d=256
flat = row_tokens.view(b*n, p, d)   # [B*20,72,256]
hidden = self.encoder(flat)         # [B*20,72,256]
logits = self.head(hidden)          # [B*20,72,200]
logits = logits.view(b,n,p,200)     # [B,20,72,200]
pred_x = soft_expected_x(logits)    # [B,20,72]

Neden B*20?

Her lane adayı ayrı bir 72 tokenlık sequence gibi decoder’a girer.

Yani 20 lane adayı, 20 ayrı kısa cümle gibi işlenir.

Transformer burada ne görüyor?

Aynı lane’in 72 row token’ı birbirini görür.

Bu sayede alt row’daki karar ile üst row’daki karar daha uyumlu hale gelebilir.

5. hidden Lane Üzerinde Neye Denk Geliyor?

hidden shape olarak [B,20,72,256]. Bu, her boncuğun transformer sonrası güncellenmiş iç hafızasıdır.

Bu vektör doğrudan çizilmez. Ama hemen ardından Linear(256 → 200) ile x kutusu skorlarına dönüşür.

6. S1 Loss Neyi Öğretiyor?

S1, S0 loss’una ek olarak token loss kullanır:

# loss_s1.py
pred = logits[bi, pred_idx]        # [matched_lanes,72,200]
tgt = targets[bi]["x_bins"][gt_idx]# [matched_lanes,72]
loss_token = cross_entropy(pred, tgt, ignore_index=-100)

x_bins nedir?

GT lane x koordinatının 200 kutudan hangisine düştüğüdür.

Örnek: x=400 px ise yaklaşık bin 100.

Lane üzerinde

Her sarı boncuk için doğru x kutusunu yüksek skorla seçmeyi öğretir.

Geçersiz row’larda -100 ile loss ignore edilir.

8. Güncel S1: Residual Modda Kaba Çizgiyi Düzeltme

Güncel S1 iki mod biliyor: direct ve residual. Residual modda row decoder sıfırdan x çizmek yerine S0/S1 kaba x skorunun üstüne düzeltme ekler.

# dynlaneseq_s1.py residual mode
coarse = self.heads(q)
coarse_x = coarse["pred_x_rows"]                  # [B,20,72]
coarse_x_norm = coarse_x / input_w
extra = self.coarse_x_embed(coarse_x_norm)        # [B,20,72,256]

row = self.row_decoder(build_row_tokens(q, extra=extra))
row_x_logits = coarse["row_x_logits"] + scale * row["row_x_logits"]
pred_x_rows = soft_expected_x(row_x_logits)

Array tarafı

coarse_x_embed, her row'daki kaba x'i 256 boyutlu token bilgisine çevirir. Yani row token artık “ben hangi y'deyim?” yanında “kaba x'im neresi?” bilgisini de taşır.

Lane tarafı

Model çizgiyi sıfırdan aramak yerine pembe taslak çizginin üstüne küçük düzeltme yapar. Bu daha stabil bir öğrenme yolu verir.

Local attention ne yapar?

RowTokenDecoder(local_attn_window) açıksa bir row sadece yakın row'lara bakar. Lane üzerinde bu, alt sıradaki boncuğun çok uzaktaki üst boncuktan değil, yakın komşularından etkilenmesi demektir.
