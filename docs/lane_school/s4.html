Bu kodda en önemli satır: stage1_x artık stage2 sampler için koordinat olur.

3. Stage1 Çıktısı İçinde Ne Var?

stage1_out aslında S3/S2 forward çıktısıdır. İçinde şunlar var:

stage1_out = {
  "coarse": {...},
  "final": {
    "row_x_logits": [B,20,72,200],
    "pred_x_rows":  [B,20,72],
    "row_hidden":   [B,20,72,256],
    ...
  },
  "evidence": {
    "sample_x_rows": [B,20,72],
    "E_seq":         [B,20,72,256],
    ...
  },
  "queries": [B,20,256],
  "features": [B,256,72,200]
}

stage1_x

İlk final çizginin x koordinatları. Görselde pembe çizgi.

row_hidden

Stage1 decoder’ın her lane-row için 256 sayılık iç hafızası. Koordinat değil, bağlam vektörü.

4. sample_x2 Nasıl Üretiliyor?

sample_x2 = self.curriculum.build_sample_x(
    stage1_x,
    targets,
    matches,
    alpha=float(sampler_beta)
)

Bu, S2’deki curriculum ile aynı fikir ama bu kez başlangıç çizgisi coarse_x değil stage1_x.

beta=1

Stage2 sampling GT’ye yakın tutulur.

beta=0.5

GT ve stage1 çizgisi karışır.

beta=0

Stage2 tamamen stage1 çizgisine güvenir.

5. evidence2 Nedir?

evidence2, ikinci kez okunan lane üstü kanıttır:

evidence2 = self.sampler(enc_features, sample_x2)

enc_features: [B,256,72,200]
sample_x2:    [B,20,72]
evidence2:    [B,20,72,256]

Tek bir vektör:

evidence2[b,n,p,:]
= stage1 çizgisi rehber alınarak,
  n'inci lane'in p'inci row noktasından ikinci kez okunan 256 feature

Yani S4’te evidence iki kez vardır:

E_seq_stage1

İlk sampling’den gelen kanıt.

E_seq_stage2

Stage1 çizgisinden sonra yapılan ikinci sampling kanıtı.

6. Stage1 Hidden Neden Stage2’ye Ekleniyor?

h1 = stage1_out["final"]["row_hidden"]  # [B,20,72,256]
if detach_stage1_hidden:
    h1 = h1.detach()

tokens2 = build_final_tokens(q, evidence2, stage_extra=hidden_scale * h1)

h1 array olarak

Stage1 decoder’ın her lane-row için öğrendiği iç temsil.

Bu pred_x_rows değildir; koordinat değil, 256 sayılık bağlamdır.

Lane üzerinde

Stage2 sadece yeni evidence’a bakmaz; stage1’in “ben bu row hakkında ne anlamıştım?” hafızasını da alır.

hidden_scale bu hafızanın ne kadar güçlü ekleneceğini belirler.

Detach edilmesi, stage2 loss’unun stage1 hidden yolundan karmaşık gradient göndermesini engeller. Bu stabilite tercihidir.

9. Güncel Not: Şu An Asıl Odak S3 Active Corridor Pipeline

Güncel config'lerde güçlü hat daha çok S3 + active_corridor + quality_calibrator + geometry_evidence etrafında dönüyor. S4 hâlâ “ikinci bakış” fikrini anlatmak için önemli; ama güncel pratikte S3 zaten lane çevresinde zoom yapıp kaliteyi kalibre ettiği için S4'ün rolü daha deneysel okunmalı.

S3 zoom

Tek stage içinde center_x → active corridor → refined_x → bridge → quality.

S4 ikinci bakış

Stage1 final çizgiyi alıp ikinci evidence okumasına koordinat yapar.
