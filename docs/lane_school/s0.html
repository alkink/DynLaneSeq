S0 derin anlatımTensor → çizgi

S0: Query vektörü ilk kez çizilebilir lane koordinatına dönüşür.

S0'ın amacı basit ama kritik: görüntüden 20 aday lane üretmek. Bu sayfada “hangi array nerede çizgi oluyor?” sorusunu takip edeceğiz.

Önemli: S0’da query hâlâ koordinat değildir. Koordinat row_x_logits → soft_expected_x → pred_x_rows yolunda ortaya çıkar.

1. S0’ın Girdi ve Çıktısı

Girdi

images shape: [B,3,288,800]. Bu gerçek yol görüntüsüdür.

Ara temsil

queries shape: [B,20,256]. Bu 20 aday lane temsilidir; koordinat değildir.

Çıktı

pred_x_rows shape: [B,20,72]. Bu artık lane çizmek için kullanılabilir x koordinatlarıdır.

2. Encoder Görüntüyü Nasıl Query’ye Bağlıyor?

# dynlaneseq_s0.py / DynLaneSeqEncoder.forward_features
feats = self.backbone(images)
fpn = self.fpn(feats)
f_proj = self.proj(fpn)
mem_value = f_proj.flatten(2).transpose(1, 2)
mem_key = (f_proj + self.pos(f_proj)).flatten(2).transpose(1, 2)
q_seed = self.queries(images.shape[0])
q1 = self.decoder(q_content, q_pos, mem_key, mem_value)

Tensor tarafı

f_proj: [B,256,72,200]. Görüntünün 72x200'lük anlam haritası.

mem_value: [B,14400,256]. Her feature konumu bir satıra açıldı.

q_seed: [B,20,256]. 20 lane adayı.

q1: [B,20,256]. Görüntüye bakmış lane adayları.

Lane tarafı

Feature map, yol görüntüsünü küçük hücrelere böler.

Query bu hücrelere bakar ve “benim aradığım lane nerede olabilir?” bilgisini içine toplar.

Henüz çizgi yok. Sadece çizgi çıkarabilecek hale gelmiş 20 aday var.

3. S0 Heads: Query Nasıl Üç Farklı Cevaba Ayrılıyor?

# heads_s0.py
exist_logits = self.exist(q)       # [B,20,2]
row_x_logits = self.row_x(q)       # [B,20,72,200]
pred_x_rows = soft_expected_x(row_x_logits) # [B,20,72]
range_raw, range_norm = self.range(q)       # [B,20,2]

exist_logits

Her slot için iki skor: lane mi, boş mu?

Görselde bu, “bu aday çizilsin mi çizilmesin mi?” kararıdır.

row_x_logits

Her lane adayı, 72 yatay satır ve 200 x kutusu için skor verir.

Görselde bu, çizginin hangi yatay pikselde geçeceğini aramaktır.

range_norm

Lane’in dikey görünür aralığıdır: başlangıç y ve bitiş y.

Görselde çizginin sadece yol üstündeki parçasını tutar.

4. row_x_logits İçinde Gerçekten Ne Var?

Bu tensoru yazılımcı gibi okuyalım:

row_x_logits[b, n, p, k]

b = batch içindeki görüntü
n = 20 lane adayından hangisi
p = 72 yatay row'dan hangisi
k = 200 x kutusundan hangisi

Örnek: row_x_logits[0, 3, 40, 120] şu demektir: “0. görüntüde, 3. lane adayı, 40. yatay satırda, x kutusu 120 ne kadar olası?”

# common.py
probs = softmax(row_x_logits)
expected_bin = sum(probs * bin_index)
x_pixel = expected_bin * (800 / 200)

800/200 = 4. Yani bin 120 civarı yaklaşık x=480 piksele denk gelir.

5. Lane Çizgisi Hangi Noktalardan Oluşuyor?

pred_x_rows[b,n] 72 tane x değeri verir. y değerleri modelden çıkmaz; sabittir:

y_rows = [0, 4, 8, 12, ..., 284]
lane_noktaları =
  (pred_x_rows[b,n,0], 0)
  (pred_x_rows[b,n,1], 4)
  ...
  (pred_x_rows[b,n,71], 284)

6. Hungarian Matching S0’da Hangi Tensorları Kullanıyor?

Training sırasında 20 tahmin var, ama gerçek lane sayısı daha az olabilir. Matcher şunu yapar:

# matcher_s0.py
p_lane = softmax(exist_logits)[..., 0]
cost_obj = -log(p_lane)
cost_point = avg_abs(pred_x_rows - gt_x_rows)
cost_range = abs(range_norm - gt_range)
cost = 2*cost_obj + 5*cost_point + 1*cost_range

Tensor tarafı

pred_x_rows tahmin edilen çizgidir.

target["x_rows"] gerçek çizgidir.

Matcher bu ikisini satır satır karşılaştırır.

Lane tarafı

Modelin mavi çizgisi gerçek sarı çizgiye yakınsa maliyet düşer.

En düşük maliyetli eşleşmeler seçilir.
