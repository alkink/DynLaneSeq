Güncel DynLaneSeq-EG: önce şeridi bul, sonra şeridin çevresinde zoom yap.

Yeni modelin ana fikri artık daha net: query önce kaba lane kartı üretir. Sonra o çizginin üstünden ve sağ-sol koridorundan feature okur. En sonda S3, bu okunan kanıtı lane’e özel filtreleyip piksel düzeltmesine yardım eder.

Bunu şöyle düşün: İlk çizgi kalemle atılmış taslak. Active corridor bu taslağın sağ-soluna küçük büyüteç koyuyor. Dynamic depthwise bridge ise büyüteçten gelen sinyali lane boyunca temizliyor.

Görsel: kendi 2k subset çıktılarından alınmış bir prediction örneği.

0. Güncel Pipeline'ı Tek Bakışta Oku

1. Feature Map Gerçek Hayatta Nasıl Düşünülür?

features [B,256,72,200] görüntünün küçültülmüş ama anlamlandırılmış haritasıdır. Piksel resmi gibi RGB değildir. Her hücrede 256 sayı vardır. Bu 256 sayı “burada çizgi kenarı var mı, yol dokusu var mı, gölge mi, şerit boyası mı?” gibi ham olmayan ipuçlarını taşır.

# feature map'i görüntü gibi değil, ipucu haritası gibi oku
features[b, :, 48, 80]  # 256 sayı
# bu hücre yaklaşık input görüntüdeki bir yol bölgesine denk gelir

2. S0 Geometry Evidence: Query İlk Taslağından Öğreniyor

Yeni S0 sadece query ile head çalıştırmıyor. İstersen önce bir taslak lane çiziyor, sonra bu taslak lane üstünden feature okuyup query'yi güncelliyor.

# dynlaneseq_s0.py
draft = heads(q)
sample_x = draft["pred_x_rows"]       # [B,20,72]
q_final, dbg = geometry_refiner(q, features, sample_x)
final = heads(q_final)

Array karşılığı

sample_x[b,n,p], n’inci lane adayının p’inci row'daki x pikselidir. Refiner bu x noktalarının altındaki feature'ları okur.

Lane karşılığı

Model “ben böyle bir şerit çizdim; şimdi bu çizginin üstünde gerçekten şerit boyası/kenar var mı?” diye tekrar bakar.

Bu, query instability sorununu tamamen çözmez. Ama query'yi boş vektör olmaktan çıkarıp kendi çizdiği geometriyle tekrar besler. Problem “query ne öğrensin?” sorusundan “taslak çizgi üstündeki kanıt query'yi nasıl düzeltsin?” formuna döner.

3. Active Corridor: S3'teki Zoom Hissi Asıl Buradan Başlıyor

Senin “S3 laneye zoomluyor ve piksel düzeltme yapıyor” sezginin kod karşılığı özellikle ActiveCorridorSearch tarafında görünüyor. Model sadece taslak x noktasına bakmıyor; x'in sağına ve soluna offset adayları koyuyor.

offsets = [-32,-24,-16,-8,0,8,16,24,32]
offset_samples = sample_local_window(features, center_x)
# [B,20,72,9,256]

weights = softmax(offset_logits)      # [B,20,72,9]
pred_delta = sum(weights * offsets)   # [B,20,72]
refined_x = center_x + pred_delta     # [B,20,72]

offset_samples[b,n,p,o,:] tek bir lane-row-offset noktasından okunan 256 sayıdır. o offset indeksidir: soldaki nokta mı, merkez mi, sağdaki nokta mı?

4. Dynamic Depthwise Bridge: Kanıt Dizisini Lane Boyunca Temizleme

Active corridor x'i düzeltmeye yardım eder. S3 bridge ise evidence [B,20,72,256] içeriğini temizler. Güncel güçlü konfigürasyonda bridge tipi dynamic_depthwise_sequence.

# dynamic_depthwise_bridge.py
params = generator(queries)          # her lane için filtre üret
e = evidence[b,n]                    # [72,256]
filtered = depthwise_conv1d(e)        # row boyunca komşu kanıtları süz
out = e + bridge_scale * delta        # shape aynı: [72,256]

Kodda ne değişiyor?

Koordinat değişmiyor. evidence[b,n,p,:] vektörlerinin 256 kanallı içeriği değişiyor.

Görüntüde nasıl düşün?

Şeridin üstündeki 72 sarı nokta bir ip gibi. Bridge bu ipin üstündeki gürültülü sinyali komşu noktalarla birlikte yumuşatıyor.

5. Quality Calibrator: “Bu Çizgiye Güveniyor muyum?”

Yeni S3 yalnızca x üretmiyor; final kararını ve kalite skorunu da sonradan kalibre edebiliyor. Bunu çizgi üretildikten sonra row hidden, confidence, offset entropy ve delta büyüklüğü gibi sinyallerle yapıyor.

stats = [
  row_conf_mean,
  active_offset_entropy,
  active_offset_max_prob,
  abs_delta,
  final_minus_coarse,
  curvature_optional,
]
delta_exist, delta_quality = quality_calibrator(stats, pooled_hidden)
final_exist_logits += delta_exist
quality_logits += delta_quality

Lane üzerinde bu şu demek: model çizgiyi çizdikten sonra “bu çizgi çok oynadı mı, koridor seçimi kararsız mıydı, row skorları güçlü mü?” diye kontrol edip skoru düzeltir.
