S3’ü anlamanın anahtarı şu: S2’deki evidence [B,20,72,256] aynen gelir. S3 bridge bu tensorün içeriğini lane query’ye göre yeniden düzenler. Shape aynı kalır; bilgi değişir.

1. S3 Görselini Tensorlara Ayıralım

Görselde bridge’i “çizgiyi biraz oynatan parça” gibi görmek cazip ama teknik olarak daha doğru cümle şudur: bridge, çizginin üstünden okunan 256 kanallı kanıt vektörlerini değiştirir.

2. S3 Kodda S2’nin Neresine Giriyor?

# dynlaneseq_s2.py akışı
evidence = self.sampler(enc["features"], sample_x)
evidence, bridge_debug = self.bridge_evidence(evidence, q)
row = self.row_decoder(self.build_final_tokens(q, evidence))

S2’de bridge_evidence sadece evidence’ı aynen döndürür:

# DynLaneSeqS2
def bridge_evidence(self, evidence, queries):
    return evidence, {}

S3 bunu override eder:

# dynlaneseq_s3.py
def bridge_evidence(self, evidence, queries):
    return self.bridge(evidence, queries)

S2

Sampler’dan gelen evidence direkt final decoder’a gider.

S3

Sampler’dan gelen evidence önce bridge’den geçer, sonra final decoder’a gider.

3. Bridge’in Girdi ve Çıktı Shape’i

input:
  evidence = [B,20,72,256]
  queries  = [B,20,256]

output:
  refined_evidence = [B,20,72,256]
  debug stats = bridge_scale, mean_abs_delta_E, ...

evidence[b,n,p,:]

n’inci lane’in p’inci row noktasından okunan 256 sayılık ham görsel kanıt.

queries[b,n,:]

n’inci lane adayının 256 sayılık kimliği. Bridge’e “bu kanıtı hangi lane için süzüyorsun?” bilgisini verir.

4. Low-Rank Bridge Parametreleri Ne?

Kodda default dim=256, rank=16, kernel_size=3. Query’den şu parametreler üretilir:

# low_rank_bridge.py
self.param_dim = dim*rank + rank*dim + rank*kernel_size
               = 256*16 + 16*256 + 16*3
               = 8240

params = self.generator(queries)  # [B,20,8240]

Yani her lane query, kendi bridge işlemi için 8240 sayı üretir. Bunlar üç parçaya bölünür:

u = vec[:4096].view(256,16)
v = vec[4096:8192].view(16,256)
s = vec[8192:].view(16,1,3)

U [256,16]

256 kanallı evidence’ı 16 kanallı dar alana indirir.

s [16,1,3]

72 row boyunca küçük 1D filtre uygular. Komşu row kanıtlarını hesaba katar.

V [16,256]

Dar 16 kanalı tekrar 256 kanala açar.

5. Bridge Tek Bir Lane İçin Ne Yapıyor?

Loop içinde her batch ve lane slot ayrı işlenir:

for bi in range(B):
  for ni in range(20):
    e = evidence[bi, ni]   # [72,256]
    q = queries[bi, ni]    # [256]
    params = generator(q)
    ...
    out[bi, ni] = e + bridge_scale * delta

Buradaki e, tek bir lane adayının 72 row boyunca okuduğu tüm evidence dizisidir:

e[0,:]  = en üst/alt row noktasındaki 256 feature
e[1,:]  = sonraki row noktasındaki 256 feature
...
e[71,:] = son row noktasındaki 256 feature

6. Low-Rank İşlemi Lane Üzerinde Nasıl Canlandırılır?

e_norm = norm(e)       # [72,256]
z = e_norm @ U         # [72,16]
z_conv = conv1d(z, s)  # [72,16]
delta = z_conv @ V     # [72,256]
out = e + scale*delta  # [72,256]

Array tarafı

[72,256] → [72,16] → [72,16] → [72,256].

Yani bilgi önce daraltılır, row boyunca filtrelenir, tekrar genişletilir.

Lane tarafı

Her sarı evidence noktası tek başına değil, komşu row noktalarıyla birlikte temizlenir.

Bu, çizgi üstündeki kanıt dizisini daha tutarlı hale getirmeye çalışır.

7. Neden “Low-Rank”?

Full bir 256→256 dönüşüm her lane için çok büyük ve pahalı olabilir. Low-rank yaklaşım, 256 kanalı 16 kanallı dar bir ara yoldan geçirir.

Bu, “büyük filtre üretme” problemini tamamen yok etmez; onu daha küçük ve kontrollü bir dönüşüme çevirir.

8. Bridge Debug Değerleri Ne Anlatır?

return out, {
  "bridge_scale": self.bridge_scale.detach(),
  "mean_abs_delta_E": mean_delta,
  "mean_abs_E_seq": mean_e,
  "delta_ratio": mean_delta / (mean_e + 1e-6),
}

mean_abs_E_seq

Ham evidence büyüklüğü. Kanıt vektörleri genel olarak ne kadar büyük?

mean_abs_delta_E

Bridge’in yaptığı değişiklik büyüklüğü. Eğer çok küçükse bridge etkisiz kalıyor olabilir.

delta_ratio

Değişiklik / ham kanıt oranı. Bridge kanıtı ne kadar oynatıyor?

bridge_scale

Delta’nın ne kadar güçlü ekleneceğini öğrenen skaler.

9. S3’ü Tek Kod Akışıyla Hatırla

evidence
  [B,20,72,256]
  S2 sampler'dan gelen ham lane üstü kanıt

queries
  [B,20,256]
  her lane adayının kimliği

params = generator(queries)
  [B,20,8240]

for each lane:
  e [72,256]
  z = e @ U       -> [72,16]
  z = conv1d(z)   -> [72,16]
  delta = z @ V   -> [72,256]
  refined = e + scale*delta

refined_evidence
  [B,20,72,256]
  shape aynı, içerik lane'e göre süzülmüş

final_x = row_decoder(q + row_emb + adapter(refined_evidence))
  [B,20,72]
