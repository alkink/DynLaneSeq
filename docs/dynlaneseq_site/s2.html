S2: Küçük kamerayı lane üstünde gezdir.

S2, modelin asıl “evidence-grounded” hale geldiği yer. Önce coarse lane çiziliyor. Sonra bu lane’in üzerindeki row noktalarından feature okunuyor. Final decoder artık sadece query’ye değil, lane üstünden gelen görsel kanıta da bakıyor.

Bunu şöyle düşün: Önce haritada yolu tahmin et. Sonra yolun üstüne küçük kameralar koy. Kameralar “burada gerçekten şerit çizgisi var mı?” diye kontrol etsin.

S2 2k subset çıktısı. Bu phase’te final çizgi, lane üstünden okunan evidence ile beslenir.

S2 Hangi Sorunu By-Pass Ediyor?

S1 row token üretir ama row token’ın elinde o satırdaki lane çevresinden doğrudan okunmuş feature yoktur. S2 bu boşluğu kapatır: “global feature bana yetmez, tahmin ettiğim eğrinin üstünden özel kanıt okuyayım” der.

Ne görüyor?

Feature map, lane query ve coarse lane tahmini.

Neye çeviriyor?

E_seq: her lane için 72 satırlık evidence dizisi.

Sonraki adıma ne gönderiyor?

Final row decoder’a query + row_embedding + evidence.

Bunu Nasıl Yapıyor?

# dynlaneseq_s2.py özü
coarse_row = row_decoder(query + row_embedding)
coarse_x = coarse_row["pred_x_rows"]

sample_x = curriculum.build_sample_x(
    coarse_x, targets, matches, alpha=sampler_alpha
)

evidence = CurveAlignedSampler(features, sample_x)
tokens = query + row_embedding + EvidenceAdapter(evidence)
final_row = row_decoder(tokens)

Curve-Aligned Sampler Ne Yapıyor?

CurveAlignedSampler, predicted lane’in x koordinatları ile sabit y row’larını birleştirir. Sonra bu koordinatları grid_sample formatına çevirir. Yani “feature map’in şu noktalarından oku” der.

# curve_aligned_sampler.py fikri
x = sample_x_rows.clamp(0, input_w - 1)
y = fixed_y_rows(num_rows)
grid = input_to_grid(x, y)
sampled = grid_sample(features, grid)
E_seq = sampled.view(batch, slots, rows, channels)
Eğer local window açıksa tek çizgi üstünden değil, çizginin sağından solundan da birkaç offset okunabilir. Config’te offset örneği: [-8, -4, 0, 4, 8].

Curriculum Neden Lazım?

Burası zor. Eğitim başında coarse lane rastgeleye yakındır. Sampler doğrudan o çizgiye güvenirse yanlış yerden evidence okur. Model yanlış kanıtla kendini daha da bozabilir.

Erken eğitim

sample_x GT lane’e yakın tutulur. Biraz noise eklenir. Model doğru lane çevresinden kanıt okumayı öğrenir.

Geç eğitim

alpha azalır. Model yavaş yavaş kendi coarse tahmininden evidence okumaya geçer.

# sampler_curriculum.py fikri
mixed = alpha * gt_x + (1 - alpha) * coarse_pred_x
sample_x = where(valid_gt_row, mixed, coarse_pred_x)
sample_x = clamp(sample_x)
sample_x = detach(sample_x)  # stabilite için

Bu problemi tamamen çözmez. Ama “yanlış tahmin -> yanlış kanıt -> daha yanlış tahmin” döngüsünü eğitim başında yumuşatır.

Evidence Adapter Ne İşe Yarıyor?

Evidence doğrudan decoder’a bastırılmaz. EvidenceAdapter küçük bir MLP ve öğrenilebilir gamma ile evidence etkisini kontrollü ekler.

# evidence_adapter.py
adapted = gamma * MLP(E_seq)
tokens = query + row_embedding + adapted
Eğer gamma hep sıfıra yakın kalırsa model evidence kullanmıyor olabilir. Eğer çok hızlı büyürse evidence branch modeli bozabilir.

Görsel Üzerinden Oku

S2’de final lane, coarse tahmin üzerinden okunan evidence ile güncellenir.

Bu görüntüleri “çizgi üstündeki küçük kamera noktaları final çizgiyi besliyor” diye oku.

S2’nin Sınırı

  • Evidence var ama henüz lane query’ye göre dinamik biçimde rafine edilmiyor.
  • Coarse prediction yanlışsa sampler hâlâ yanlış bölgeye kayabilir.
  • Topology yok; bağlantılar hâlâ doğrudan modellenmiyor.
