S1: Lane’i tek çizgi değil, boncuk dizisi gibi üret.

S1, S0’daki doğrudan row head yerine RowTokenDecoder kullanır. Her lane slotu için 72 row token oluşur. Her token “bu satırda x nerede?” sorusunu cevaplar.

Token = küçük karar parçası. Burada her token bir y satırındaki x konumunu arayan parçadır.

S1 2k subset görseli. Şerit artık row token dizisi üzerinden üretilir.

S1 Neyi Farklı Yapıyor?

S0’da

Bir query vektörü, tek MLP head ile 72 row’un x-bin skorlarını üretir.

S1’de

Query her row için ayrı token’a yayılır. Transformer encoder bu 72 token’ın birbirini görmesini sağlar.

Görsel olarak: S0 “tek kalem darbesiyle çiz” gibidir. S1 “ipi 72 boncuğa böl, boncukları sırayla ayarla” gibidir.

Soft Expected X Neden Bu Kadar Önemli?

argmax “en yüksek skoru seç” demek. Eğitimde argmax kullanırsan modelin hatası seçimin arkasındaki skorlara düzgün geri akmaz. Bu yüzden kod soft_expected_x kullanır.

Argmax gibi düşünme

bin = argmax(logits)
x = bin * bin_width
loss(x, gt_x)  # gradient zayıf/kopuk

Soft expected gibi düşün

prob = softmax(logits)
x = sum(prob * bin_centers)
loss(x, gt_x)  # gradient akar

Bu aslında “gürültülü çizgiyi temizleme” için yumuşak bir kontrol sağlar. Model sadece doğru kutuyu seçmeyi değil, yakın kutular arasında düzgün dağılım kurmayı da öğrenir.

Loss Tarafı

S1, S0 loss’una token loss ekler. Yani hem koordinat düzgün olsun ister, hem de doğru x-bin sınıfı yüksek skor alsın ister.

# loss_s1.py fikri
base = S0Loss(exist + point + range + smooth)
loss_token = cross_entropy(row_x_logits, target_x_bins)
loss_total = base + w_token * loss_token
Bu iki hedef birbirini tamamlar: point loss çizgiyi piksel olarak yaklaştırır, token CE doğru bin’i netleştirir.

S1 hâlâ lane üstünden görsel evidence okumaz. Query görüntüye attention ile bakmıştır, ama final row decoder’ın elinde doğrudan “bu row’da lane üstünden aldığım local feature” yoktur. Bu yüzden S2’ye ihtiyaç var.
